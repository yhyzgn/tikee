# Phase4 P2：任务拓扑二级页面与画布全屏

## 背景
任务拓扑抽屉已经具备 SVG 画布、跨工作流影响分析和明细表，但抽屉承载空间不足，不适合后续图形交互和事故复盘扩展。

## 目标
- 将任务拓扑从 Jobs 页面抽屉迁移为独立二级页面 `/jobs/topology`。
- Jobs 页面仅保留入口按钮，点击后路由跳转，不再加载拓扑数据或渲染拓扑抽屉。
- 二级页面承载完整画布、影响分析、依赖/引用/unresolved 明细和返回任务列表操作。
- 画布区域支持全屏/退出全屏切换，便于复杂拓扑查看。

## 实现约束
- 保持拓扑 API 不变：`GET /api/v1/jobs/topology`、`GET /api/v1/jobs/{job}/impact`。
- 保持 `TopologyCanvas` 独立组件，避免 JobsPage 继续膨胀。
- 全屏为前端布局能力，不改变浏览器 Fullscreen API 权限模型；通过固定定位 overlay 实现。

## 验收
- `/jobs/topology` 路由可访问，并受登录态保护。
- Jobs 页面“任务拓扑”按钮跳转到二级页面。
- 二级页面展示拓扑图形画布、跨工作流影响分析和拓扑明细。
- 画布支持“全屏 / 退出全屏”切换。
- 前端 lint/build/test 通过。
